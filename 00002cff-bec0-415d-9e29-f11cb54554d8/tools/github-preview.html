Help & resolved URL
Supported input URL forms:
  • https://github.com/<owner>/<repo>/blob/<ref>/<path>
  • https://github.com/<owner>/<repo>/tree/<ref>/<path> — treated as directory
  • https://github.com/<owner>/<repo>/raw/<ref>/<path>
  • https://raw.githubusercontent.com/<owner>/<repo>/<ref>/<path> (or with refs/heads/ prefix)
Multi-path probing: for a request like /path/to/file, the service worker tries in order: /path/to/file, /path/to/file.html, /path/to/file/index.html, /path/to/file/index.htm. A trailing slash (/path/to/dir/) forces directory mode (only tries index.html/index.htm inside it).

Authentication: public requests use raw.githubusercontent.com. When a PAT is set, requests use the GitHub Contents API (api.github.com) with Accept: application/vnd.github.raw, which is the documented path for authenticated raw access.

Limitations:
  • Page must be served over http://localhost or https:// — service workers don't run on file://.
  • Pages that load assets via absolute root paths (<img src="/logo.png">) won't resolve, because the service worker scope is limited to this directory.
  • Branch/ref names containing slashes must be URL-encoded in the input or the bare-form raw URL.
  • Default-branch detection from a bare repo URL is not performed; include a branch in the URL.
PAT security: tokens are stored in sessionStorage (cleared when this tab closes) by default. Checking "Remember" moves them to localStorage — convenient but readable by any script on this origin. Use a fine-grained token scoped to the minimum repos.
